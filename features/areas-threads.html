How it works

Structure that matches
how the work actually splits

Not a blank page, not another wiki. One proven shape, three levels deep, that maps to how professional responsibility really accumulates.

Areas: one plate per responsibility.

Each thing you are responsible for gets its own area, so parallel work never blurs together. Status reads at a glance: active, in review, blocked, or stable, each with its own quiet colour.

Threads: a living line of work.

Inside an area, threads carry each strand from kickoff to done. A thread is open while it moves, on hold when it waits, and done when it lands. The area stays tidy because the work lives in threads.

Entries: the atomic record.

Every thread is a timeline of entries: an update, a todo, a decision, a meeting, a blocker. Each one typed, dated, and colour-coded, so the history of any line of work reads itself back to you.
